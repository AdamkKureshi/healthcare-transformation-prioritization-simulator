### 4.1 Expected Monetary Value (EMV)

Expected Monetary Value (EMV) is used to estimate the expected transformation value of an initiative under uncertain implementation conditions.

The simplified analytical formula used in the dashboard is:

EMV = (Success Probability × Success Impact)

+ (Partial Success Probability × Partial Success Impact)

+ (Failure Probability × Failure Impact)

Net EMV = EMV − Estimated Cost

The objective of EMV modeling is to simulate how transformation offices may compare competing investments according to expected strategic value rather than optimistic assumptions alone.

The methodology assumes that healthcare transformation initiatives may produce:
- full transformation success
- partial implementation success
- unsuccessful or low-impact implementation outcomes

Each outcome is associated with:
- a probability estimate
- a simulated financial or operational impact value

The EMV framework therefore introduces:
- uncertainty-aware prioritization
- risk-adjusted transformation analysis
- investment comparison logic
- probabilistic transformation thinking

The dashboard recalculates Scenario EMV dynamically when scenario controls are adjusted, allowing users to explore how changing implementation assumptions influence portfolio attractiveness.

### 4.6 Monte Carlo Simulation

The platform applies Monte Carlo simulation techniques to model portfolio uncertainty under probabilistic implementation conditions.

The simulation repeatedly samples transformation outcomes using:
- success probabilities
- partial success probabilities
- failure probabilities
- simulated impact values

Each simulation run generates a potential portfolio outcome based on randomly selected implementation scenarios.

Repeated simulation runs allow the dashboard to estimate:
- expected portfolio value ranges
- downside exposure
- upside potential
- variability across implementation environments
- portfolio resilience under uncertainty

The Monte Carlo framework introduces probabilistic transformation modeling concepts commonly used in:
- financial risk analysis
- investment modeling
- portfolio governance
- strategic uncertainty analysis

The simulation outputs currently include:
- outcome distributions
- minimum simulated outcomes
- maximum simulated outcomes
- percentile ranges
- average portfolio value

The current implementation uses simplified illustrative assumptions.

A real-world implementation may potentially incorporate:
- historical implementation performance
- implementation delay distributions
- transformation adoption variability
- operational disruption probabilities
- governance uncertainty factors
- expert-estimated transformation outcomes

The Monte Carlo methodology is intended to demonstrate how healthcare transformation planning may evolve beyond deterministic planning approaches into uncertainty-aware portfolio governance environments.

### 5.2 Scenario Controls

Allows modification of transformation success probability assumptions.

The scenario adjustment slider dynamically recalculates transformation attractiveness across the portfolio.

Increasing the slider value simulates:
- more favorable implementation conditions
- stronger governance coordination
- higher transformation success likelihood
- improved portfolio outcomes

Decreasing the slider value simulates:
- implementation uncertainty
- operational instability
- governance friction
- lower transformation success probability

As the slider is adjusted, the dashboard automatically updates:
- Scenario Net EMV
- portfolio rankings
- risk positioning
- portfolio attractiveness
- transformation prioritization outcomes

This functionality is intended to demonstrate how transformation portfolios may behave under changing operational and implementation environments.

---

### 5.3 Available Portfolio Budget

Simulates financial constraints and prioritizes initiatives under a constrained funding envelope.

The Available Portfolio Budget control dynamically changes which initiatives may be selected within the portfolio.

Increasing the available budget:
- allows selection of more initiatives
- expands portfolio coverage
- enables inclusion of higher-cost transformation programmes

Decreasing the available budget:
- forces stricter prioritization
- reduces portfolio breadth
- emphasizes comparatively high-value initiatives

As the budget value changes, the dashboard automatically recalculates:
- selected initiatives
- total selected portfolio cost
- portfolio transformation value
- investment allocation tradeoffs

This functionality is intended to simulate how healthcare transformation offices, donor programmes, and PMOs may prioritize investments under real-world funding constraints.
